Habit Tracker MCP Server

A Model Context Protocol (MCP) server implementation that enables AI assistants like Claude to track, store, and analyze user habits and daily activities through intelligent conversation.

Node.js MCP SDK LowDB Claude AI Integration
View on GitHub

Project Purpose

This project serves as a foundational implementation to understand the Model Context Protocol (MCP) structure. It bridges the gap between AI reasoning capabilities and local data persistence, demonstrating how AI agents like Claude can be extended with custom tools to manage personal data effectively.

Key Features

  • Conversational Logging

    Log habits naturally: "I drank 2 liters of water today" or "Read 25 pages". The AI automatically parses and stores structured data.

  • Persistent Storage

    Uses LowDB for a transparent, lightweight JSON-based local database that survives restarts.

  • Smart Analysis

    Ask Claude "Show me my recent habits" or "How is my exercise this week?" to retrieve and analyze your data on demand.

  • Timestamped Entries

    Automatic recording of precise timestamps for every entry, enabling time-based tracking and habit streak analysis.

Technical Architecture

MCP Server

Implements the standard I/O transport layer to communicate with Claude Desktop. Registers add_habit and get_habits tools.

Database Layer

Abstraction layer over LowDB (`db.js`) that manages the `data.json` file schema and asynchronous read/write operations.

Tool Schema Example 

// add_habit Tool Definition
{
  name: "add_habit",
  description: "Adds a new habit or daily activity",
  inputSchema: {
    type: "object",
    properties: {
      name: { type: "string", description: "Activity name" },
      amount: { type: "number", description: "Numerical value" },
      unit: { type: "string", description: "Unit (e.g., 'Liters')" }
    },
    required: ["name", "amount", "unit"]
  }
}

Technology Stack

  • LowDB
