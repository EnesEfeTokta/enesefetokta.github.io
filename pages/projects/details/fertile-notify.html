View Repository

Platform Overview

**Fertile Notify** is a robust, scalable notification platform designed to handle event-driven notifications across multiple communication channels. Built with modern software architecture principles, it provides a centralized solution for managing and delivering notifications to users.

The system receives events from external applications, processes notification rules based on user subscriptions and templates, and delivers messages asynchronously using background workers with built-in retry mechanisms and failure handling.

Core Capabilities

  • Event-Driven Ingestion

    RESTful API for receiving events with validation and processing.

  • Async Processing

    Non-blocking notification delivery via background workers.

  • Multi-Channel

    Support for Email (SMTP), SMS, and In-App notifications.

  • Retry Mechanism

    Automatic retries with exponential backoff for failed deliveries.

  • Template Engine

    Dynamic templates with placeholder substitution per event type.

  • Subscription Plans

    Flexible plans (Free, Basic, Premium) with usage limits.

API Usage Example

Sending a notification is as simple as POSTing an event payload. 

POST /api/notifications
{
  "userId": "user-123",
  "eventType": "user_registered",
  "payload": {
    "userName": "John Doe",
    "email": "john@example.com",
    "registrationDate": "2024-01-15"
  }
}

// Response
{
  "message": "Event received and queued for processing",
  "eventId": "evt-456"
}
